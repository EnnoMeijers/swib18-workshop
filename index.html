Compacting

Mapping JSON keys to URIs with a @context
"name": "http://schema.org/name"

Here: context generated from ontologies

Context can do more than simple name-URI mapping

Field 'contribution' should be an array

 

"contribution": {
  "@id": "http://id.loc.gov/ontologies/bibframe/contribution",
  "@container": "@set" // <-- array
}

CORS

Error: not allowed by Access-Control-Allow-Origin

Client (our HTML+JS) is in a different location than the API (Elasticsearch server)

API server needs to set header: Access-Control-Allow-Origin

In production: specify which domain can make requests

Exercise 16: Configure CORS in Elasticsearch


"http": {
    "cors.enabled": true,
    "cors.allow-origin": "*"
}

(in /etc/elasticsearch/elasticsearch.yml, then: sudo service elasticsearch restart)
